# Canadian Tax Advisor — Build Context

## Product
TaxReady — Canadian Tax Prep Guide (V1)
KYC-style interview tool generating a personalized T1 preparation brief.
Output: checklist only — no tax computation.
Target: Ontario residents, newcomers, first-time filers.
Tax year: 2025 (filed spring 2026).

## Stack
- Next.js 14, TypeScript, Tailwind CSS
- Railway (nixpacks, npm run start, no Dockerfile)
- AI: OpenRouter API, model via env var AI_MODEL
- Default dev model: meta-llama/llama-3.3-8b-instruct:free
- No database — static JSON files only
- tsx runner (not ts-node) — ESM config requirement
- Unit tests: node --import tsx --test

## interview-flow.json Architecture (CRITICAL)
The interview flow is split into per-branch source files:
  taxonomy/2025/branches/00_life_events_gate.json  through  15_credits_review.json

interview-flow.json is AUTO-GENERATED by scripts/buildInterviewFlow.ts
It is NEVER hand-edited. It is the assembled build artifact.

This split-file architecture was adopted after a single large file (424KB) proved
unmanageable — copy-paste corruption introduced ~60+ structural and text errors
that could not be reliably detected or repaired.

## Folder Structure

## Confidence Tiers
confirmed — all eligibility conditions met
possible  — partial/fact_dependent/condition_not_asked/professional_judgment_required
(two tiers only — no third tier)

## Urgency Tiers
standard  — bring to preparer
priority  — required to claim this item
urgent    — time-sensitive, act before deadline
critical  — irreversible consequence if missed

## Output Brief Structure (5 sections)
1. Income to Report
2. Deductions to Claim
3. Credits to Apply
4. Documents to Gather (tiered: critical → urgent → priority → standard)
5. Obligations and Actions
Plus: Planning Discussion items (preparer modelling required)

## AI Clarification Layer
Endpoint: /api/clarify
Accepts: { node_id, ita_section, user_question, plain_language }
Returns: { explanation: string }
Usage: clarification popups per ITA section — not tax advice
Server-side only — API key never exposed to client
