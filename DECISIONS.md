# TaxReady — Key Decisions Log

## Product Naming
- Brand: TaxReady
- Product V1: Canadian Tax Prep Guide
- V2 may become: Canadian Tax Advisor
- Domain: check TaxReady.ca availability

## Product Scope — MVP
- Federal T1 + Ontario only
- Ontario credits: OTB (OTA s.103.1), LIFT (OTA s.103.4), Senior Homeowners Grant (OTA s.104.1)
- Checklist output only — no tax computation
- 2025 tax year (filed spring 2026)
- Target: Ontario residents, newcomers, first-time filers
- AI layer: clarification popups only, not advice

## Interview Architecture
- 16 entries: branch_life_events_gate (order 0) + 15 branches
- always_runs: true for branches 0, 1, 2, 3, 13, 14, 15
- triggered_by flags for branches 4–12
- Fixed branch order — dynamic ordering is v2
- Back button uses replaySession for clean undo
- Multi-select feedback iterates all selected values

## interview-flow.json File Architecture (CRITICAL)
- Single 413KB monolith was abandoned — copy-paste corruption introduced 60+ errors
  that were not reliably fixable by automated patching (two failed attempts)
- NEW: each branch lives in its own file: taxonomy/2025/branches/NN_branch_id.json
- Per-branch files are ~15–25KB each — fully readable and writable by any LLM in one pass
- interview-flow.json is AUTO-GENERATED by: npx tsx scripts/buildInterviewFlow.ts
- interview-flow.json is NEVER hand-edited directly
- Each branch can be independently validated, tested, and regenerated
- Build script: scripts/buildInterviewFlow.ts
  - reads all .json files in taxonomy/2025/branches/ in alphabetical order
  - validates branch_id, branch_label, branch_order, screens present
  - warns on non-sequential branch_order values
  - supports --year flag for multi-year builds

## Output Brief Structure
1. Income to Report
2. Deductions to Claim
3. Credits to Apply
4. Documents to Gather (urgency-tiered: critical/urgent/priority/standard)
5. Obligations and Actions
Plus: Planning Discussion items

## Confidence Tiers
- confirmed: all eligibility conditions met
- possible: partial/fact_dependent/condition_not_asked/professional_judgment_required
- No third tier — possible implies seek professional review

## Urgency Tiers
- standard:  bring to preparer
- priority:  required to claim item
- urgent:    time-sensitive — act before deadline
- critical:  irreversible consequence if missed

## Education Branch — Multi-Student Household
The "who is the student" question uses multi_select with independent flags:
- taxpayer_is_student
- dependent_is_student
tuition_transfer_decision flag activates when dependent_is_student, triggering
a preparer planning note for transfer vs carryforward modelling.

## T2200 Decoupled from Work-From-Home
t2200_required flag is activated by any employment expense screen (home office,
vehicle, supplies). It is not exclusive to the work-from-home answer.

## Safety Deposit Box — Removed
Not deductible since the 2013 Federal Budget (ITA s.20(1)(bb) amended effective 2014).
This option does not appear anywhere in the interview.

## Partnership — Self-Employment Branch
Partnership income is reported via T2125 or T5013. Partnership is an explicit
option in Branch 7 alongside sole proprietor, incorporated, and freelance/gig.

## Gate Screen Pattern (UX Sprint — COMPLETE)
Multi-select gate screens were introduced in Branch 3 and Branch 14 as part of the
April 2026 UX sprint to eliminate sequential yes/no screens for simple filers.

Pattern: gate screen asks "did any of the following apply?" with exclusive "None of
the above" option. Detail content is delivered inline as gate feedback. Downstream
screens (where a follow-up is genuinely required) carry show_if keyed to the gate
question_id using the "contains" operator.

| Branch | Gate screen_id | question_id | Detail delivery |
|--------|---------------|-------------|-----------------|
| 3 — Other Income | other_income_gate | other_income_types | CPP/OAS, employer pension, foreign pension, EI, rental: inline gate feedback. RRSP withdrawal, support received, investments: separate screens with show_if contains gate value. |
| 14 — Deductions Review | deductions_gate | deductions_applicable | Charitable, medical, carrying charges, support paid: inline gate feedback. No separate detail screens. deductions_prior_year_noa always shown (not gated). |
