# TaxReady — Key Decisions Log

## Product Naming
- Brand: TaxReady
- Product V1: Canadian Tax Prep Guide
- V2 may become: Canadian Tax Advisor
- Domain: check TaxReady.ca availability

## Product Scope — MVP
- Federal T1 + Ontario only
- Ontario credits: OTB (OTA s.103.1), LIFT (OTA s.103.4), Senior Homeowners Grant (OTA s.104.1)
- Checklist output only — no tax computation
- 2025 tax year (filed spring 2026)
- Target: Ontario residents, newcomers, first-time filers
- AI layer: clarification popups only, not advice

## Interview Architecture
- 16 entries: branch_life_events_gate (order 0) + 15 branches
- always_runs: true for branches 0, 1, 2, 3, 13, 14, 15
- triggered_by flags for branches 4–12
- Fixed branch order — dynamic ordering is v2
- Back button uses replaySession for clean undo
- Multi-select feedback iterates all selected values

## interview-flow.json File Architecture (CRITICAL DECISION)
- Single 424KB file was abandoned — copy-paste corruption introduced 60+ errors
  that could not be reliably detected or repaired through testing or LLM patching
- NEW: each branch lives in its own file: taxonomy/2025/branches/NN_branch_id.json
- interview-flow.json is AUTO-GENERATED by scripts/buildInterviewFlow.ts
- interview-flow.json is NEVER hand-edited
- Per-branch files are ~25KB each — fully readable/writable by any LLM in one pass
- Each branch can be validated, tested, and regenerated independently

## Taxonomy
- Single taxonomy node schema (see CONTEXT.md)
- items.json is array of TaxonomyNode
- Both versioned by tax year: taxonomy/2025/
- Validation: npx tsx src/lib/validateTaxonomy.ts
- Test runner: node --import tsx --test

## Output Brief Structure
1. Income to Report
2. Deductions to Claim
3. Credits to Apply
4. Documents to Gather (urgency-tiered: critical/urgent/priority/standard)
5. Obligations and Actions
Plus: Planning Discussion items

## Confidence Tiers
- confirmed: all eligibility conditions met
- possible: partial/fact_dependent/condition_not_asked/professional_judgment_required
- No third tier — possible implies seek professional review

## Urgency Tiers
- standard:  bring to preparer
- priority:  required to claim item
- urgent:    time-sensitive — act before deadline
- critical:  irreversible consequence if missed

## Education Branch — Multi-Student Household (DECISION)
The "who is the student" question uses multi_select with three independent flags:
- taxpayer_is_student
- spouse_is_student
- dependent_is_student
This handles all 7 household combinations cleanly.
When spouse_is_student OR dependent_is_student activates alongside another student
flag, tuition_transfer_decision flag fires, triggering a planning discussion item.
Childcare deduction school-attendance exception (ITA s.63) flagged when
spouse_is_student and children are present.

## T2200 Decoupled from Work-From-Home
T2200 is required for ANY employment expense claim (home office, vehicle, supplies,
musical instruments). The t2200_required flag is activated by any employment expense
screen, not only the work-from-home screen.

## Safety Deposit Box — Removed
Safety deposit box fees are NOT deductible. The 2013 Federal Budget eliminated
this deduction effective the 2014 tax year (ITA s.20(1)(bb) amended).
This option has been removed from the Deductions Review branch.

## Dependents — Profile Screen Trigger
Branch 6 (Children and Dependents) triggers on has_dependents flag which is set
by a question in Branch 1 (Personal Profile). This captures taxpayers who had
dependents before 2025 and did not select life_event_had_child at the gate.
Dependents include: children (any age if still dependent), parents/grandparents,
other financially supported family members.

## Partnership — Self-Employment Branch
Partnership income (general or limited) is reported on T1 via T5013 or T2125.
Added as an option in Branch 7 alongside sole proprietor and incorporated.
